void act ( char *  action_message,
int  hide_invisible,

OBJ_DATA *  obj,

)

Show an action relative to the actors and watchers.

Parameters
[in]action_messageA specially formatted action string.
[in]hide_invisibleWhen set do not show the action to those who cannot see the actor.
[in]chThe primary actor.
[in]objA target object or character.
[in]vict_objAn additional target object or character
[in]typeBitvector defines who is to see this action.
Local Variables:
  • strp - This is an index into the action_message parameter.
  • point - This is an index into the buf local.
  • i - text to be inserted in place of format tags.
  • p - A pointer into the buf local during the reformat phase.
  • buf - The output buffer.
  • immbuf1 - A buffer holding the actor's real name.
  • immbuf2 - A buffer holding the victim's real name.
  • chsex - A temporary backup of a PC's sex if they are hooded.
  • color - A reference to the ANSI color to use for output part.
  • do_cap - Determines proper capitalization (start of line, e.g).
  • dietyCap - Capitalize the actor's god name (wtf?).
  • totemCap - Capitalize the actor's totem name (again: w.t.f.?).
  • to - A pointer to the recipient of the action message.
  • tch - A pointer to an included secondary actor.
Implementation:

  • First we test the parameter integrity as a precaution. If we have bad parameters we return immediately.

  • We return immediately if the N/PC is in Combat "Compete" Mode.
    See Also
    do_compete()

  • Examine type to determine who receives this action message.

  • Iterate through persons in the recipient's room sending the action message where appropriate.

    • We do not translate and send the message:

      • IF the recipient is not the actor
      • AND the actor is immortal
      • AND the recipient is mortal

      • AND the actor is invisible.

    • We translate and send the message:

      • IF the recipient is connected (i.e. a link-live PC or an animated NPC)
      • AND the recipient is not the actor (OR this message type is to the actor)
      • AND the recipient is in the actor's group (OR this message type is not group-only)
      • AND the recipient can see the actor (OR we do not want to hide invisible actors
      • AND the recipient is awake
      • AND the type is not immortal-only (OR the recipient is on staff and the actor is not (?))
      • AND NOT a non-victim message IF the recipient is the same as the victim.

    • Iterate through the action_message and output buffer character-at-a-time.

      • We parse the next character as a format identifier IF this character is $. ELSE we copy this character from the message to the buffer (stopping when we reach the end of the string.

      • Format translations:

        • $n - insert the short description of actor (showing "someone" if the recipient cannot see, and storing the actor's name for immortals.

        • $N - insert the short description of the victim (showing "someone" if the recipient cannot see, and storing the victim's name for immortals to see.

        • $3 - insert the short description of third PC (?).

        • $m - actor as "him", "her", or "it" (the latter in the case of neuter or hooded actors).

        • $M - victim as "him", "her", or "it" (the latter in the case of neuter or hooded victims).

        • $s - actor as "his", "her", or "its" (the latter in the case of neuter or hooded actors).

        • $S - victim as "his", "her", or "its" (the latter in the case of neuter or hooded victims).

        • $e - actor as "he", "she", or "it" (the latter in the case of neuter or hooded actors).

        • $E - victim as "he", "she", or "it" (the latter in the case of neuter or hooded victims).

        • $o - object name or "something" if hidden from the recipient

        • $O - victim object name or "something" if hidden from the recipient

        • $p - object short description or "something" if hidden from the recipient

        • $P - victim object short description or "something" if hidden from the recipient

        • $a - "a" or "an" object (based on name)

        • $A - "a" or "an" victim object (based on name)

        • $T - Cast parameter vict_obj as a string and set to local i (?).

        • $F - Cast parameter vict_obj as a string of names and set to local i to the first of them (?).

        • $g - insert the name of the actor's god (?)

        • $G - insert the name of the victim's god (?).

        • $$ - Insert a dollar-sign

      • Do not copy local i to buf IF there is nothing to insert (OR we are at the end of the buffer), ELSE Capitalize, color, and copy.

    • Cap the end of the output buffer with a newline & null.

    • We do not send the buffer:

      • IF this is a combat message
      • AND the recipient filters combat messages
      • AND the recipient is not fighting the victim
      • AND the recipient is not fighting the actor
      • AND the victim is not fighting the recipient
      • AND the actor is not fighting the recipient

    • We do not send the buffer:

      • IF this is a firefight message
      • AND the recipient filters firefight messages
      • AND the recipient is not fighting the victim
      • AND the recipient is not fighting the actor
      • AND the victim is not fighting the recipient
      • AND the actor is not fighting the recipient

    • We do not send the buffer:

      • IF this is a bleeding message
      • AND the recipient filters combat messages
      • AND the recipient is not fighting the victim
      • AND the recipient is not fighting the actor
      • AND the victim is not fighting the recipient
      • AND the actor is not fighting the recipient
      • AND the recipient is fighting

    • We do not send the buffer:

      • IF this is a search/scan message
      • AND the recipient filters combat messages
      • AND the recipient is grouped or admin/li>

    • Reformat the buffer IF the type parameter is set accordingly

    • Send the buffer to the recipient

    • IF the action type is to a single recipient, we can stop iterating. ELSE process the next recipient.

  • Return to the caller after the message has been sent to any recipients
